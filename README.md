## Project Overview
The goal of this project is to explore, evaluate, and improve web accessibility using three approaches:
- Manual testing  
- Automated testing  
- AI-assisted analysis  

The project is divided into multiple phases across several weeks, each focusing on different accessibility techniques and tools.


## Project Phases

### Weeks 1–2: Manual Accessibility Testing
- Compare “Before” vs “After” versions of demo pages  
- Perform:
  - Keyboard navigation testing  
  - Screen reader testing  
  - Color contrast analysis  
  - Forms accessibility testing  
- Document accessibility issues  
- Create manual test cases

### Weeks 3–4: Automated Accessibility Testing
- Setup Playwright project with TypeScript  
- Integrate axe-core  
- Run accessibility scans on all pages  
- Export results (JSON)  
- Implement end-to-end test flows  
- Generate custom accessibility reports  

### Weeks 5–6: AI-Assisted Accessibility Testing
- Integrate OpenAI API  
- Run AI-based accessibility analysis:
  - Screenshot analysis  
  - axe-core results summarization  
  - AI-generated fix suggestions  
- Evaluate AI accuracy and usefulness  

### Weeks 7–8: Final Project
- Define a problem in accessibility testing  
- Build an AI-based solution (e.g. assistant, analyzer, reporting tool)  
- Prepare demo and presentation  
- Compare:
  - Manual vs Automated vs AI approaches  

## 📁 Repository Structure
- `/Manual Testing` – Reports, issues, and comparisons, manual test cases  
- `/Automation` – Playwright + axe-core setup and tests, generated reports (JSON, Markdown, etc.)  
- `/AI` – AI experiments and outputs  
- `/Final Project` – Final solution and demo materials
